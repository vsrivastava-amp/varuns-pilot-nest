PUBLISHER CIV (pCIV) EXTRACTION — V1

Define the commercial intent (pCIV) of the FULL conversation (all prior turns + the reply just written) as JSON per SCHEMA; omit missing/null fields. Output ordering/delimiting per the output protocol above.

=== SCHEMA ===

{"commercial":bool,"type":"discovery|investigational|transactional|executional","topic":str,"targets":[{"gpc":[int],"sellers":[str],"price_min":num,"price_max":num,"price_target":num,"price_currency":str,"product_name":str,"brand":str,"gender":"male|female|unisex","age_group":"newborn|infant|toddler|kids|adult","condition":"new|used|refurbished|open_box|collectible"}]}

commercial : true if any commercial relevance; false → omit type, topic, and targets.
topic      : one-line summary of the user's query (max 10 words). Omit when commercial:false.
targets    : cap at 5 total. See TARGET RULES.

=== TYPE RULES ===

discovery        Learning WHAT something is / HOW it works / WHETHER it's worth it.
                 → "is GORE-TEX worth it"

investigational  Deciding WHICH product/brand — recommendation-seeking or comparing options ("best for X", "help me pick", named comparisons).
                 → "Dyson V15 vs Shark Stratos"

transactional    Ready to buy — seeking availability, price, or fulfillment.
                 → "cheapest PS5 near me"

executional      Delegating purchase to an agent to execute.
                 → "buy me everything to start beekeeping"

commercial:false ONLY when zero connection to any commercial topic (math, geography, creative writing).
Any question about a product, service, technology, travel, or lifestyle → commercial:true.

=== TARGET RULES ===

1. One target from the USER QUERY (what the user explicitly asked for).
2. Additional targets from the ASSISTANT RESPONSE — one per specific brand/product named that was NOT in the user query. Cap total at 5.

For discovery: target may contain only gpc (no product_name required) when no specific product is named.
gpc           : one or more L2 IDs; use multiple when the product spans categories (e.g. gaming headset→[223,1294]). L1 if no L2 exists. Omit for pure brand-only queries or when the query is too broad to imply a category (e.g. "gift ideas for X"). Do not assign GPC based on recipient age or relationship alone.
sellers       : only when user explicitly names a store. Never infer.
price_min/price_max/price_target/price_currency : "under $X"→price_max; "over $X"→price_min; "around $X"→price_target.
product_name  : type stripped of brand AND condition words ("refurbished Nike running shoes"→"running shoes"; "used iPhone 15"→"iPhone 15").
brand         : from query or assistant response; infer well-known implicit brands (iPhone→Apple).
condition     : used/pre-owned/vintage→used; refurbished/renewed→refurbished; open box→open_box. Omit if not stated — never add condition:"new".
age_group     : baby/infant→infant; toddler→toddler; kids/children/teen or age under 18→kids. Only from explicit user words.
gender        : men's/for men→male; women's/for women/ladies→female. Only from explicit user words.
